1. Node 1 — Page Classifier

N1 페이지 로드 · dataLayer Listener 주입 · 페이지 타입 분류
1
Playwright 브라우저 실행
target_url로 Chromium 브라우저를 열고 페이지를 로드합니다. GTM_AI_HEADLESS 환경변수가 1이면 headless(창 없음), 기본은 headed(창 표시)로 실행됩니다.
2
Persistent Listener 주입 (inject_listener)
page.add_init_script()로 JavaScript를 주입합니다. 이 스크립트는 window.dataLayer.push를 후킹해서 모든 push 이벤트를 window.__gtm_captured 배열에 자동으로 누적합니다. 페이지를 이동해도 리스너가 유지됩니다.
3
초기 dataLayer 이벤트 수집
window.__gtm_captured를 읽어 페이지 로드 시점에 이미 발화된 이벤트(예: page_view, view_item)를 수집합니다. 각 이벤트는 {data, timestamp, url} 형태로 저장됩니다.
4
LLM으로 페이지 타입 분류
HTML 스냅샷을 LLM에 전달해 page_type을 판별합니다: PLP PDP cart checkout home unknown
5
dataLayer 상태 결정
수집 이벤트 수로 상태를 결정합니다: 3개 이상 → full (N1.5 생략), 1~2개 → partial, 0개 → none (N1.5 실행)
▼ 입력 변수
target_url접속할 URL
▲ 출력 변수
page_typePLP/PDP/cart/checkout/home/unknown
datalayer_statusfull / partial / none
datalayer_events_found발견된 이벤트명 목록
current_url브라우저 현재 URL

Listener 주입 코드 구조

// window.__gtm_captured에 모든 dataLayer.push 이벤트 누적
const _originalPush = window.dataLayer.push.bind(window.dataLayer);
window.dataLayer.push = function() {
  for (var i = 0; i < arguments.length; i++) {
    var item = arguments[i];
    window.__gtm_captured.push({
      data: JSON.parse(JSON.stringify(item)),  // 깊은 복사 (나중 변조 방지)
      timestamp: Date.now(),
      url: window.location.href
    });
  }
  return _originalPush.apply(window.dataLayer, arguments);
};

이벤트 source 필드로 트리거 타입 자동 결정

6. Node 6 — GTM Creation (핵심: GTM API 호출)

N6 plan 설계안을 GTM API로 실제 생성: Variable → Trigger → Tag (순서 엄수)

전체 실행 흐름

Step A: Workspace 확보

Workspace 분기 로직
state에 workspace_id 있음 → 목록 조회·HITL·생성 전부 생략. 해당 ID 바로 사용
기존 워크스페이스 수 < 3 → gtm-ai-{timestamp} 이름으로 신규 생성
기존 워크스페이스 수 ≥ 3 → HITL로 재사용 여부 확인 (reuse 선택 시 gtm-ai-* 중 최신 사용)
429 Rate Limit (최대 3회) → 30/60/90초 대기 후 재시도 → 실패 시 기존 gtm-ai-* 재사용

Step B: plan 자동 보정 (_fix_plan)

"""
_fix_plan이 하는 3가지 보정:
1. captured_events의 DL 이벤트마다 CE - {event_name} 트리거가 없으면 자동 생성

3. 잘못된 firing_trigger_names 참조를 정상 이름으로 교체

Step C: Variable 생성

for var_spec in plan["variables"]:
    variable = _build_variable(var_spec)    # GTMVariable 객체로 변환

_build_variable: type별 처리
type == "v" (DLV) → GTMVariable(name, type="v", parameters=[dataLayerVersion, name]) 그대로
type == "d" + ID 있음 → GTMVariable(type="d", [elementId, attributeName]) — HTML id 기반
type == "d" + CSS selector만 있음 → 자동으로 type="jsm"으로 변환 (CSS selector → Custom JavaScript)
type == "jsm" (CJS) → GTMVariable(type="jsm", [javascript 함수 코드])
type == "js" (LLM 오타) → "jsm"으로 자동 보정 (_VARIABLE_TYPE_MAP)
값이 비어 있음 → None 반환 → 상위에서 드롭 (생성 안 함)

Step D: Trigger 생성 (이름→ID 매핑 저장)

for trig_spec in plan["triggers"]:
    trigger = _build_trigger(trig_spec)
    result = client.create_or_update_trigger(workspace_id, trigger)
    created_triggers.append(result)
    trigger_name_to_id[result["name"]] = result["triggerId"]  # 이름 → ID 맵
customEventFilter 강제 보정: LLM이 arg0에 DLV 변수 참조를 넣는 경우가 있습니다. _fix_custom_event_filter가 arg0를 항상 {"{{_event}}"}로 강제합니다. GTM API는 customEventFilter의 arg0가 반드시 {{_event}}여야 하기 때문입니다.

Step E: Tag 생성 (trigger 이름 → ID 변환)

for tag_spec in plan["tags"]:
    firing_names = tag_spec.get("firing_trigger_names", [])
    # "CE - view_item" 이름을 실제 triggerId로 변환
    firing_ids = [
        trigger_name_to_id[name]
        for name in firing_names
        if name in trigger_name_to_id
    ]
    tag = _build_tag(tag_spec, firing_ids)
    result = client.create_or_update_tag(workspace_id, tag)
    created_tags.append(result)

7. GTM API 호출 상세 (GTMClient)

API 호출 패턴 (모든 리소스 공통)

def create_or_update_variable(self, workspace_id: str, variable: GTMVariable) -> dict:
    # 1. 기존 동일 이름 리소스 조회
    existing = self._find_resource_by_name("variables", workspace_id, variable.name)
    if existing:
        # 2. 이미 있으면 update (덮어쓰기)
        return self._update("variables", workspace_id, existing["variableId"], body)
    else:
        # 3. 없으면 create (신규 생성)
        return self._create("variables", workspace_id, body)

9. State 전달 흐름 — GTMAgentState

LangGraph는 모든 노드 간 데이터를 GTMAgentState TypedDict 하나로 전달합니다. 각 노드는 자신이 담당하는 필드만 갱신한 새 dict를 반환하고, 나머지 필드는 이전 state에서 그대로 이어받습니다.

# 각 노드 반환 패턴
return {
    **state,               # 이전 state 전체 복사 (변경되지 않은 필드 유지)
    "workspace_id": workspace_id,          # N6 갱신
    "created_variables": created_variables, # N6 갱신
    "created_triggers": created_triggers,   # N6 갱신
    "created_tags": created_tags,           # N6 갱신
    "error": None,                          # 성공 시 None
}

핵심 라우팅 분기점

라우팅 함수분기 기준 필드경로
route_after_classifier datalayer_status == "full" true → N2, false → N1.5 → N2
route_after_active_explorer cart_addition_events 있음 true → N3.25, false → N3.5 or N4
route_after_explorer manual_required 비어있음 true → N5, false → N4 → N5
route_after_planning plan_approved && error 없음 true → N6, false → N8 (Reporter)
route_after_creation error 없음 true → N7 (Publish), false → N8
