## 1. 프로젝트 개요

**GTM AI Agent**는 사용자가 자연어로 GA4·이커머스 등 태깅 요구를 입력하면, **Playwright**로 실제 웹 페이지를 열어 `dataLayer` 이벤트를 수집하고, **LangGraph** 기반 멀티 에이전트가 **GTM Variable / Trigger / Tag**를 설계·생성한 뒤 **Google Tag Manager API**로 반영하고 **Publish**까지 시도하는 **자동화 에이전트 시스템**입니다.

- **입력**: 대상 URL, 자연어 요청, GTM Account/Container ID(및 선택적 Workspace)  
- **출력**: GTM 컨테이너 내 리소스 생성, 실행 로그(`logs/<run_id>/`), 마크다운 보고서(`report.md`), 로컬 웹 UI에서 진행 상황 확인  

**대상 사용자**: 본인 GTM 계정을 사용하는 마케터·개발자(단일 사용자 도구 성격).

## 4. 주요 실행 흐름 (요약)

1. **Page Classifier** — 페이지 로드, dataLayer 진단, 페이지 유형 분류  
2. **Structure Analyzer** (조건부) — dataLayer 불완전 시 DOM/구조 보강 분석  
3. **Journey Planner** — 캡처할 이벤트·탐색 순서 계획  
4. **Active Explorer** — Navigator 루프로 클릭/탐색하며 이벤트 캡처  
5. **Manual Capture** (조건부) — 자동 불가 이벤트에 대한 게이트웨이  
6. **Planning · HITL** — 설계안 생성 및 사용자 승인( CLI / UI )  
7. **GTM Creation** — Variable → Trigger → Tag API 생성  
8. **Publish** — 버전 생성 및 Publish(권한·스코프에 따라 수동 안내 가능)  
9. **Reporter** — `report.md` 및 토큰 사용 요약  

데이터는 `utils/ui_emitter`를 통해 `logs/<run_id>/state.json`, `events.jsonl` 등으로 기록되며, UI는 이를 폴링해 타임라인·리포트를 표시합니다.

### 4.1 UI·상태 동기화 (문서·코드 기준)

| 항목 | 설명 |
|------|------|
| **Resources vs Report** | GTM 생성 실패 시에도 LangGraph `state`로 `report.md`는 작성될 수 있으므로, `gtm_creation`이 **`state.json`에 `created_*`·`workspace_id`를 성공/실패 모두 기록**해 두 탭이 어긋나지 않게 함. |
| **타임라인** | 그래프 분기로 호출되지 않은 노드가 `queued`로 남지 않도록, Reporter 진입 시 **`reconcile_timeline_at_reporter`** 등으로 `skip`/`failed` 정리. |
| **Live 화면** | 타임라인 클릭 시 상세 패널이 `running` 노드에 고정되지 않도록 선택 로직 수정; `node_enter` 기준으로 Thought를 **노드별 필터**. |
| **종료 스냅샷** | `runner`가 완료 시 **`current_node=8`** 을 기록해 기본 하이라이트가 의미 있게 유지됨. |

## 5. 실행 방법 (제출 채점·재현용)

### 5.1 준비

```bash
cd gtm_ai
pip install -r requirements.txt
playwright install chromium
```

### 5.2 환경 변수

- 프로젝트 루트에 `.env`를 두고 **`OPENAI_API_KEY`** 를 설정합니다.  
- **제출 압축본에는 `.env`가 포함되지 않습니다.** 대신 `.env.example`을 참고해 로컬에서 생성하세요.

### 5.3 GTM OAuth (실제 GTM 연동 시)

```bash
python gtm/auth.py
```

- 발급된 토큰은 일반적으로 `credentials/` 아래에 저장됩니다.  
- **제출본에는 `credentials/` 폴더 및 민감 JSON이 포함되지 않습니다.** 채점자는 본인 OAuth 클라이언트로 재인증하면 됩니다.

### 5.4 UI 서버

```bash
python serve_ui.py
```

브라우저에서 `http://localhost:8766/ui/` (포트는 `serve_ui.py`의 `PORT` 참고) 로 접속합니다.

### 5.5 CLI

```bash
python main.py
```

## 7. 과제·프로젝트에서의 역할 요약

- **문제 정의**: 수동 GTM 설정에 드는 시간과 dataLayer·이벤트 매핑 오류를 줄이기 위한 자동화.  
- **해결 접근**: 관측 가능한 브라우저 상태와 API를 묶는 **명시적 상태 그래프(LangGraph)** 와 **HITL**로 안전장치 확보.  
- **검증**: 실제 이커머스 URL에 대해 이벤트 캡처·플랜 생성·GTM 반영까지의 **엔드투엔드 파이프라인**으로 검증(로그는 로컬 `logs/`에 남기며, 제출본에서는 제외).
